These are some things you might want to include in task prompts (from my email briefs task, customize as needed):

To prevent the task from running even when not necessary:

"ONLY BOTHER RUNNING IF IT IS BETWEEN THE HOURS OF 8AM and 5PM. EITHER RUN BECAUSE IT IS WITHIN THE WINDOW OR SKIP CHECKING, DO NOT HEDGE OR ASK IF YOU SHOULD CHECK."

(And don't forget to schedule your tasks after creating them. Conditions in task prompts are only evaluated if the task runs in the first place.)

To prevent being notified of things when it runs:

"Do not mention processed emails (even urgent sounding ones) in your response. Just reply saying the processing was completed, unless you ran into something broken while working on it- that is worth reporting."

It is also very helpful (and cheaper) to include any skills the task will need when it runs in the skills field. Those skills are automatically loaded when the task runs so the agent doesn't have to go look for them on its own.
